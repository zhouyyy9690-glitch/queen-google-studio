# AI 协作开发规则（Stable Mode）

本项目进入“稳定开发模式”。在后续开发中必须严格遵守以下规则，优先级高于任何功能设计建议。

## 一、最高原则
本项目的唯一目标是：**稳定推进 narrative game 内容开发**。禁止构建通用引擎或过度优化架构。

## 二、绝对禁止行为
1. **禁止主动重构**：禁止提出系统重构建议。
2. **禁止引入新层级**：禁止引入新的 DSL、新抽象层或新的系统架构（如 evaluator registry, script system 等）。
3. **禁止修改核心架构**：禁止修改 `GameContext`、`scene system`、`UI 结构`或 `flags 存储结构`。
4. **禁止批量修改**：禁止一次性迁移或重构旧剧情数据。
5. **禁止过度优化**：禁止“顺手优化代码结构”或跨模块重写逻辑。

## 三、允许行为
1. **内容层开发**：编写剧情、修改单个 scene、使用已存在 DSL（Condition / Action）。
2. **局部修复**：修复明确命名的 bug 或单个函数问题，确保不影响其他系统。
3. **最小扩展**：仅在用户明确要求时增加字段或逻辑。

## 四、设计限制
任何设计必须：
- 不影响现有系统结构。
- 不引入新框架。
- 保持“人类可读”且“AI 生成友好”。
- 可以在单文件或局部实现。

## 五、开发方式
- **NL-driven**：用户用自然语言描述需求，AI 负责最小化完成实现。
- **系统冻结**：Condition DSL、Action DSL、GameContext、UI 结构视为冻结状态，仅可使用，不可重做。

## 六、执行优先级
1. 不破坏现有系统。
2. 保持游戏可运行。
3. 完成当前剧情或功能任务。
4. 不执行非必要的架构优化。
